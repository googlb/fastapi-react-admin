# Product Guidelines: FastAPI React Admin

## 1. 整体设计基调

平台应传达出**专业和高效**的整体基调。这意味着界面设计应注重清晰度、响应速度和逻辑性，以支持用户快速准确地完成管理任务。

## 2. 视觉设计原则

秉持**极简主义设计**原则。我们将充分利用 Ant Design 的简洁风格，减少不必要的视觉装饰。通过 Tailwind CSS 精确控制间距、字体大小和颜色，确保内容呈现清晰，信息传达高效。目标是创建一个既不失优雅又能突出功能性的界面。

## 3. 界面文案与沟通风格

采用**直接和简洁**的文案风格。所有界面文本，包括标签、提示信息、按钮文本和错误消息，都应使用清晰、无歧义、无术语的语言。目标是让用户无需额外思考即可理解信息，从而提高操作效率。

## 4. 用户交互反馈

提供**即时且明确的反馈**。系统应在用户执行操作后立即提供清晰的结果反馈，无论是成功、失败还是警告。我们将利用 Ant Design 的通知组件（如 `message`、`notification`）在界面上以显著且不干扰用户当前工作流的方式，提供实时、位置明确、内容清晰的反馈。

## 5. 导航与工作流设计

遵循**直观的层级导航**原则。设计清晰的菜单结构和提供面包屑导航，使用户能够轻松理解当前所处位置和返回路径。工作流设计应尽可能减少用户的认知负担和操作步骤，确保高效且顺畅的用户体验。
